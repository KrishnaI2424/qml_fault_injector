# qml_fault_injector
qml_fault_injector is a Pennylane based wrapper that provides classes and methods that inject quantum faults into a trained QML model, simulating the effects of noise. What qml_fault_injector has that's more unique is support for evaluating fault tolerance at HPC scale. THe core interface is wrapping a given QNode with a FaultConfig, that specifies fault type, probability, target wires, and device type. Then, simply running the wrapped node, with trained weights and biases, will result in an appropriately faulted circuit. There are two simulation backends that are supported: an exact density-matrix mode, using a default.mixed device, or a stochastic mode, with pure state simulators including both lightning.qubit and lightning.gpu. Because features like qml.BitFlip and qml.PhaseFlip are not avaiable in pure state simulators, the wrapper uses approximations. These approximations are accurate when the number of wires is small enough. The results are logged in ResultsRecorder, which can store and display the resulting data from injection trials.
